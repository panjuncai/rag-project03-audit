# rag-project03-audit 学习计划

这个项目适合按“先跑通，再拆解，再改造”的方式学习。它本质上是一个 RAG 合规审计系统：前端 React 页面驱动流程，后端 FastAPI 提供接口，核心链路是：

```text
PDF 加载 -> 文档解析/分块 -> Embedding -> 向量库索引 -> 相似度搜索 -> 大模型生成 -> 搜索效果评估
```

## 第 1 阶段：理解项目全貌，约 1 天

先阅读这些文件：

- `README.md`
- `backend/main.py`
- `frontend/src/App.jsx`
- `frontend/src/components/Sidebar.jsx`

需要搞清楚三件事：

1. 前端有哪些页面：加载文件、文件分块、嵌入、索引、搜索、生成、评估。
2. 后端有哪些 API：`/load`、`/chunk`、`/embed`、`/index`、`/search`、`/generate`、`/evaluate`。
3. 项目数据如何落盘：
   - `backend/01-loaded-docs`
   - `backend/01-chunked-docs`
   - `backend/02-embedded-docs`
   - `backend/04-search-results`
   - `backend/05-generation-results`

这一阶段不要急着改代码，目标是画出一张系统流程图。

这一阶段目标是让系统完整动起来，不需要一次性理解所有代码。

## 第 3 阶段：学习后端 RAG 主流程，约 3-5 天

按下面顺序阅读后端服务代码。

### 1. 文档加载

文件：

- `backend/services/loading_service.py`

重点：

- 如何读取 PDF。
- `pymupdf`、`pypdf`、`unstructured`、`pdfplumber` 的区别。
- `page_map` 是如何生成的。
- 加载后的 JSON 文件如何保存到 `01-loaded-docs`。

### 2. 文档分块

文件：

- `backend/services/chunking_service.py`

重点：

- 固定长度分块。
- 段落分块。
- 句子分块。
- 分块结果中的 `content` 和 `metadata` 如何组织。

### 3. 向量化

文件：

- `backend/services/embedding_service.py`

重点：

- `EmbeddingConfig` 的作用。
- OpenAI、HuggingFace、Bedrock 等 embedding provider 如何被封装。
- embedding 结果如何保存到 `02-embedded-docs`。

### 4. 向量库索引

文件：

- `backend/services/vector_store_service.py`
- `backend/utils/config.py`

重点：

- Milvus collection 如何创建。
- index type 如何配置。
- 向量、文本、metadata 如何一起写入向量库。

### 5. 相似度检索

文件：

- `backend/services/search_service.py`

重点：

- query 如何转成 embedding。
- `top_k` 的作用。
- `threshold` 的作用。
- 检索结果如何过滤、排序和保存。

### 6. 大模型生成

文件：

- `backend/services/generation_service.py`

重点：

- search results 如何拼接成 prompt。
- OpenAI、DeepSeek、HuggingFace 生成逻辑的区别。
- 生成结果如何保存到 `05-generation-results`。

### 7. 搜索效果评估

文件：

- `backend/services/evaluation_service.py`
- `backend/routers/evaluation.py`

重点：

- 评估 CSV 的输入格式。
- 如何批量执行搜索。
- 如何计算或整理评估结果。

建议每读完一个 service，就回到 `backend/main.py` 找对应 API，看它从请求到返回的完整路径。

## 第 4 阶段：学习前端如何调用后端，约 2-3 天

按业务页面阅读：

- `frontend/src/pages/LoadFile.jsx`
- `frontend/src/pages/ChunkFile.jsx`
- `frontend/src/pages/EmbeddingFile.jsx`
- `frontend/src/pages/Indexing.jsx`
- `frontend/src/pages/Search.jsx`
- `frontend/src/pages/Generation.jsx`
- `frontend/src/pages/Evaluation.jsx`

学习重点：

1. 每个页面调用哪个 API。
2. 请求参数是什么。
3. 后端返回什么数据。
4. 返回数据如何展示到页面上。

建议边读边整理下面这张表：

| 页面 | API | 输入 | 输出 | 对应后端服务 |
| --- | --- | --- | --- | --- |
| 加载文件 | `/load` | PDF、加载方式 | loaded document JSON | `LoadingService` |
| 文件分块 | `/chunk` | 文档 ID、分块方式、chunk size | chunked document JSON | `ChunkingService` |
| 文件嵌入 | `/embed` | 文档 ID、provider、model | embeddings JSON | `EmbeddingService` |
| 向量索引 | `/index` | embedding 文件、向量库、索引模式 | collection/index 结果 | `VectorStoreService` |
| 相似度搜索 | `/search` | query、collection、top_k、threshold | search results | `SearchService` |
| 生成回答 | `/generate` | query、模型、搜索结果 | LLM answer | `GenerationService` |
| 搜索评估 | `/evaluate` | CSV、collection、top_k、threshold | evaluation results | `EvaluationService` |

## 最重要的学习主线

学习这个项目时，不要一开始陷进所有依赖包。先抓住主线：

```text
用户上传 PDF
-> 后端读取 PDF
-> 文档被切成 chunks
-> chunks 被转成 embeddings
-> embeddings 写入 Milvus
-> 用户输入问题
-> 问题被转成 embedding
-> 从 Milvus 找相似 chunks
-> 把 chunks 和问题交给大模型
-> 生成带上下文的回答
```

主线清楚了，其他库只是工具箱。
